리액트에 대해 알려면 기본적인 문법에서(JSX 등)부터도 시작할 수 있지만, <br>
리액트 앱의 최소 구성 단위인 **'리액트 컴포넌트'**부터 이해하는 게 <br>
가장 **직관적**으로 접근할 수 있다고 생각한다.  <br><br>
컴포넌트가 무엇인지, 리액트 컴포넌트는 어떻게 기능하는지부터 알아보자.

***

>"컴포넌트"란?

CS(comeputer science)에서 컴포넌트(Component)란,<br>
**독립적인 기능을 수행하는 단위 모듈**을 의미한다.

컴포넌트를 통해 UI를 **개별적인 여러 조각**으로 나누고, 
<br>
이로인해 우리는 각 조각들을 개별적으로, 독립적으로 다룰 수 있게 된다. 
<br> 
(=> 재사용 & 유지 보수가 용이해짐)


>"리액트 컴포넌트"란?

리액트 컴포넌트는, <br>
리액트로 만들어진 앱을 이루는 최소한의 단위이다.

리액트 컴포넌트에서 다루는 데이터는 2가지, 
props와 state이다.<br>
리액트 컴포넌트는 props와 state를 입력받아 리액트 DOM 노드를 생성한다. 


## props (property) 란?
- 정의 : 부모 컨포넌트가 자식 컴포넌트에게 주는 값
- 단방향 형식 : 자식 컴포넌트는 props를 받아오기만 할 뿐, 부모 컴포넌트에게 전달할 수 없다. 또한 받아온 props를 직접 수정할 수도 없고, 안에 있는 값을 꺼내서 사용할 수만 있다.

***Example :***

```javascript
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
} 

const element = <Welcome name="Sooyeon" />;
ReactDOM.render(
  element,
  document.getElementById('root')
);

``` 

Welcome이라는 컴포넌트가 자신의 부모 컴포넌트인 element 컴포넌트에 담겨있고, 그 안에서 name="Sooyeon" 이라는 데이터를 내려받고 있다.

Welcome(props)에는 name="Sooyeon"이 담겨있는 것이고, 우리는 props.name으로 안에 있는 name의 값 "Sooyeon"을 꺼내 사용할 수 있다.

부모 -> 자식으로의 단방향 형식이기 때문에 Welcome은 element에게 거슬러 정보를 전달하진 못한다. 또한 받아온 props 내부의 데이터 name="Sooyeon"을 변경할 수 없고, 값을 그대로 꺼내서 사용할 수만 있는 것이다.
<br><br>


## state 란?

- 정의 : 컴포넌트의 특정 상태를 기억하여 화면에 반영하고, 상태가 사용자에 의해 변경되면 다시 화면이 변경되는 기능을 하기위해 존재하는 객체
- 형식 : props와 달리 컴포넌트 내부에서 선언하며, 내부에서 값을 변경할 수 있다. 
- 주의 : state 객체를 사용하기 위해서는 부모를 상속받는 class를 생성하여, 생성자로 사용해야 한다.
