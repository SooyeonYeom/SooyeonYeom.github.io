>"JSX"

**JSX = 'JavaScript + XML'**

JSX는 JavaScript와 XML를 합쳐서 만든 JavaScript의 확장문법이다.

때문에 마크업 언어(HTML XML 등)에 익숙한 우리들은 친숙한 냄새가 나는 JSX에 적응하기 쉽다! 

그렇다면 JSX로 인해 무얼 할 수 있냐? 

JSX를 사용하면 기존처럼 마크업(HTML)과 로직(JS)를 분리하는 대신, **둘 다를 포함하는 컴포넌트**를 구성할 수 있게 된다. 즉 JavaScript 코드 안에서 UI 관련 작업을 할 때 시각적으로 아주 도움이 된다는 것이다. 

참고로 리액트에서 JSX 사용은 필수가 아니나 (순수 자바스크립트로도 작성할 수 있다) 
편하게 쓰라고 만들어진 문법을 잘 써먹으면 좋으니 익숙해지도록 하자!

1) JSX의 중괄호 안에 모든 JavaScript 표현식을 넣을 수 있다.

```JSX
const name = 'Sooyeon Yeom';
const element = <h1>Hello, {name}</h1>;

ReactDOM.render(
  element,
  document.getElementById('root')
);
```


2) JavaScript 함수 호출결과를 JSX의 엘리먼트에 포함시킬 수도 있다.

```JSX
function formatName(user) {
  return user.firstName + ' ' + user.lastName;
}

const user = {
  firstName: 'Sooyeon',
  lastName: 'Yeom'
};

const element = (
  <h1>
    Hello, {formatName(user)}!
  </h1>
);

ReactDOM.render(
  element,
  document.getElementById('root')
);
```


3) JSX를 if 구문 및 for loop 안에 사용하고, 변수에 할당하고, 인자로서 받아들이고, 함수로부터 반환할 수도 있다. (= 자바스크립트 함수로서 호출 가능하며, 자바스크립트 객체로 인식된다.)

```JSX
function getGreeting(user) {
  if (user) {
    return <h1>Hello, {formatName(user)}!</h1>;
  }
  return <h1>Hello, Stranger.</h1>;
}
```



4) JSX로 속성 정의 : "따옴표" & {중괄호}

따옴표를 이용해 "문자열"을, 중괄호를 이용해 {자바스크립트 표현식}을 삽입할 수 있다.

```JSX
const element = <a href="https://www.reactjs.org"> link </a>;

const element = <img src={user.avatarUrl}></img>; //*camelCase
```

(*JSX는 HTML보다는 JavaScript에 가깝기 때문에, React DOM은 camelCase 프로퍼티 명명 규칙을 사용한다.)
